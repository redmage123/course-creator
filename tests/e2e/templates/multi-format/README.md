# Multi-Format Organization Templates

This directory contains example organization templates in multiple formats that the AI Assistant can parse to create complete organizations with training programs, tracks, courses, instructors, and students.

## üìã Available Formats

The AI Assistant supports the following template formats:

### 1. YAML Format (`.yaml`)
**File**: `organization_simple.yaml`

**Advantages**:
- Human-readable with clean hierarchical structure
- Supports comments for documentation
- Widely used in configuration management
- Easy to edit manually

**Best For**: Configuration files, DevOps workflows, documentation

**Example**:
```yaml
organization:
  name: "TechEd Academy"
  slug: "teched-academy"
  description: "Modern technology education academy"

  admin:
    full_name: "Dr. Emily Chen"
    email: "admin@teched-academy.edu"
```

### 2. JSON Format (`.json`)
**File**: `organization_simple.json`

**Advantages**:
- Native JavaScript format
- Widely supported across programming languages
- Strict validation
- Easy API integration

**Best For**: API payloads, web applications, data interchange

**Example**:
```json
{
  "organization": {
    "name": "TechEd Academy",
    "slug": "teched-academy",
    "description": "Modern technology education academy"
  }
}
```

### 3. XML Format (`.xml`)
**File**: `organization_simple.xml`

**Advantages**:
- Industry-standard for enterprise systems
- Strong schema validation support (XSD)
- Excellent for complex hierarchical data
- Self-documenting with semantic tags

**Best For**: Enterprise integrations, SOAP APIs, legacy systems

**Example**:
```xml
<organization>
  <name>TechEd Academy</name>
  <slug>teched-academy</slug>
  <description>Modern technology education academy</description>
</organization>
```

### 4. CSV Format (`.csv`)
**Files**:
- `csv/organization.csv` - Organization and admin details
- `csv/instructors.csv` - Instructor list
- `csv/students.csv` - Student list
- `csv/tracks.csv` - Learning track definitions
- `csv/courses.csv` - Course definitions

**Advantages**:
- Universal spreadsheet compatibility (Excel, Google Sheets)
- Easy bulk data management
- Simple import/export
- Non-technical users can edit easily

**Best For**: Bulk data imports, spreadsheet users, data migration

**Example**:
```csv
name,slug,description,domain
TechEd Academy,teched-academy,Modern technology education academy,teched-academy.edu
```

**Note**: CSV format uses multiple files for different entity types due to the tabular nature of CSV.

### 5. Excel Format (`.xlsx`)
**File**: `organization_simple.xlsx` (generated by `create_spreadsheet_templates.py`)

**Advantages**:
- Multiple sheets in one file
- Styled headers and formatting
- Formula support
- Familiar interface for business users
- Data validation

**Best For**: Business users, complex datasets, formatted presentations

**Structure**:
- Sheet 1: Organization - Main organization details and admin
- Sheet 2: Instructors - List of instructors
- Sheet 3: Students - List of students
- Sheet 4: Tracks - Learning track definitions
- Sheet 5: Courses - Course definitions mapped to tracks

### 6. LibreOffice Calc Format (`.ods`)
**File**: `organization_simple.ods` (generated by `create_spreadsheet_templates.py`)

**Advantages**:
- Open-source alternative to Excel
- Cross-platform compatibility
- Open document standard
- Same structure as Excel

**Best For**: Open-source environments, cross-platform workflows

**Structure**: Same as Excel (.xlsx) format

### 7. Plain Text Format (`.txt`)
**File**: `organization_simple.txt`

**Advantages**:
- No special software required
- Human-readable structured format
- Works with any text editor
- Simple parsing logic

**Best For**: Quick edits, version control, minimal tooling environments

**Example**:
```
ORGANIZATION DETAILS
===============================================================================

Name: TechEd Academy
Slug: teched-academy
Description: Modern technology education academy focusing on software development
```

## üìä Template Structure

All templates contain the same organizational structure:

### 1. Organization Details
- **Basic Info**: Name, slug, description, domain, industry, size
- **Address**: Street, city, state, postal code, country
- **Contact**: Phone, email, website
- **Administrator**: Full name, username, email, password, role
- **Settings**: Timezone, language, currency, date format

### 2. Training Programs
- Name, description, category, difficulty, duration, price, published status

### 3. Learning Tracks
- Name, description, target audience, prerequisites
- Learning objectives, duration, difficulty, max students

### 4. Courses
- Track assignment, title, description, difficulty, duration
- Note: Courses are mapped to specific tracks

### 5. Instructors
- Personal info: First name, last name, full name
- Account: Username, email, password
- Professional: Specialties, bio

### 6. Students
- Personal info: First name, last name, full name
- Account: Username, email, password
- Educational: Background, learning goals

## üîê Security Considerations

### Passwords in Templates

All template examples use placeholder passwords:
- Admin: `AdminPass123!`
- Instructors: `InstructorPass123!`
- Students: `StudentPass123!`

**Important**:
- These are **example passwords only**
- Never use these passwords in production
- Replace with secure passwords before creating real organizations
- Consider using password generation services

### Unique Identifiers

The API workflow script automatically adds timestamps to ensure uniqueness:
- Organization slug: `teched-academy-1234567890`
- Domain: `test1234567890.teched-academy.edu`
- Emails: `admin@test1234567890.teched-academy.edu`
- Usernames: `echen1234567890`

This prevents conflicts during testing and development.

## üìù Template Validation

Before using a template, ensure:

1. **Required Fields**: All mandatory fields are populated
   - Organization: name, slug, domain, admin details
   - Users: full_name, username, email, password, role

2. **Field Formats**:
   - Slug: lowercase, hyphens only (e.g., `teched-academy`)
   - Email: valid email format
   - Phone: minimum 10 characters
   - Username: alphanumeric and underscores only (pattern: `^[a-zA-Z0-9_]+$`)
   - Password: meets complexity requirements (8+ chars, mixed case, numbers, symbols)

3. **Relationships**:
   - Courses reference valid track names
   - All emails are unique
   - All usernames are unique
   - Organization domain is unique

## üõ†Ô∏è Customization Guide

### Adding New Organizations

1. **Choose Your Format**: Pick the format you're most comfortable with
2. **Copy the Template**: Duplicate one of the example files
3. **Update Organization Info**:
   - Change name, slug, domain to your organization
   - Update address and contact information
   - Set administrator details
4. **Customize Training Content**:
   - Modify tracks to match your curriculum
   - Update courses for each track
   - Adjust durations and difficulty levels
5. **Add Users**:
   - List your instructors with their specialties
   - Add initial students if needed
6. **Validate**: Ensure all required fields are filled

### Adding More Instructors or Students

**CSV Format** (easiest):
1. Open `csv/instructors.csv` or `csv/students.csv`
2. Add new rows with user information
3. Ensure all columns are filled

**JSON/YAML Format**:
1. Add new objects to the `instructors` or `students` array
2. Follow the existing structure
3. Maintain consistent formatting

**Excel/ODS Format**:
1. Open the spreadsheet
2. Navigate to the Instructors or Students sheet
3. Add new rows below the existing data
